====================ИНСТРУКЦИЯ ПО ЗАПУСКУ====================

Содержимое:
-папки project_2 и cryptoBill это сервера для обращения к АПИ.
-папка flutter это и есть flutter)

Запуск:
1)Необходимо скачать Android Studio
2) Открываем проект , слева сверху нажимаете на виджет меню ->Settings->Language & Frameworks->Flutter , и во вкладке Flutter SDK path указываете путь до flutter(папка которая есть в репозитории).
3) В тех же настройках находите Android SDK , там должен быть указан путь C:\Users\$USERNAME$\AppData\Local\Android\Sdk , до SDK.
4) Теперь необходимо создать виртуальное устройство , справа сбоку нажимаем Device Manager и нажимаем создать Virtual Device.
5) Проект создавался на эмуляторе Pixel 9 pro , чтобы выбрать такой же у вас должен быть установлен API Level 35+, его можно посмотреть нажав на виджет меню ,Project Structures, и выбрать в SDK нужный API Level.Если у вас ничего нету в SDK, то необходимо перейти в Android SDK -> SDK Tools , и выбрать в Android SDK Build-Tools нужный API Level и установить его.
6) После этого запускаете виртуальное устройство , сверху по середине можно выбрать как запустить проект, через веб или мобильный эмулятор.
7) Нажимаем на mobile и запускаем проект.
8) Для того чтобы обращаться к API запускаем сервер project_2 и cryptoBill(открываем проекты и прописываем в терминале python main.py у обоих соответсвенно).
9) Готово

При запуске виртуального устройство у меня возникла проблема в том ,что оно не хотело открываться вообще, проблема была в том что эмулятор запускался на Vulkan, а на моем компе не было этого граф. api ,поэтому если у вас такая же проблема ,необходимо в корневой папке Android studio перейти в .android и создать там ini файл в котором прописать
Vulkan = off
GLDirectMem = on
